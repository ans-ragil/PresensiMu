# Plan: Deploy PresensiMu ke Vercel + Railway

## Problem Statement

Sistem saat ini menggunakan Express server persistent + SQLite. Vercel tidak bisa menjalankan Express server, sehingga muncul 404. Solusi: **split deployment** — Frontend di Vercel, Backend di Railway (gratis).

## Architecture Setelah Deploy

```
presensimu.vercel.app (React SPA - static)
        │
        │ API calls ke backend URL
        ▼
presensimu-api.up.railway.app (Express + PostgreSQL)
        │
        │ Prisma ORM
        ▼
   PostgreSQL (Railway managed)
```

## Implementation Steps

### Step 1: Deploy Backend ke Railway

Railway mendukung Express server + PostgreSQL secara native.

**Actions:**
1. Buat akun di [railway.app](https://railway.app) (gratis $5 credit/bulan)
2. Create New Project → Deploy from GitHub repo (`ans-ragil/PresensiMu`)
3. Tambah PostgreSQL service ( Railway provides managed PostgreSQL)
4. Set environment variables di Railway dashboard:
   ```
   DATABASE_URL=<Railway auto-generated PostgreSQL URL>
   JWT_SECRET=<your-secret-key>
   JWT_EXPIRES_IN=30m
   REFRESH_TOKEN_EXPIRES_IN=7d
   NODE_ENV=production
   CORS_ORIGIN=https://presensimu.vercel.app
   ```
5. Set build command: `cd server && npm install && npx prisma generate && npm run build`
6. Set start command: `cd server && node dist/app.js`
7. Railway akan kasih URL seperti `presensimu-api.up.railway.app`

**Database Migration:**
```bash
# Setelah Railway PostgreSQL ready, run migration locally:
cd server
npx prisma migrate deploy
npm run db:seed
```

### Step 2: Update Frontend API Configuration

**File:** `client/src/services/api.ts`

Saat ini `baseURL: '/api'` — ini hanya works jika backend di belakang proxy yang sama. Untuk split deployment, perlu diubah ke environment variable.

```typescript
const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL || '/api',
  headers: { 'Content-Type': 'application/json' }
});
```

**File to create:** `client/.env.production`
```
VITE_API_URL=https://presensimu-api.up.railway.app/api
```

**File:** `client/.env.example`
```
VITE_API_URL=http://localhost:5000/api
```

### Step 3: Update Backend CORS

**File:** `server/src/app.ts`

Update CORS agar menerima request dari Vercel domain:
```typescript
app.use(cors({
  origin: [
    'http://localhost:5173',
    'https://presensimu.vercel.app',
    process.env.CORS_ORIGIN
  ].filter(Boolean),
  credentials: true
}));
```

### Step 4: Add Vercel Configuration

**File to create:** `vercel.json`

```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

Ini memastikan React Router (SPA) bisa handle semua routes.

### Step 5: Deploy Frontend ke Vercel

**Actions:**
1. Login ke [vercel.com](https://vercel.com) dengan GitHub
2. Import repository `ans-ragil/PresensiMu`
3. Set configuration:
   - **Framework Preset:** Vite
   - **Root Directory:** `client`
   - **Build Command:** `npm run build`
   - **Output Directory:** `dist`
4. Set environment variable:
   ```
   VITE_API_URL=https://presensimu-api.up.railway.app/api
   ```
5. Deploy
6. Vercel akan kasih URL seperti `presensimu.vercel.app`

### Step 6: Post-Deploy Verification

1. Buka `https://presensimu.vercel.app`
2. Login dengan akun default: `admin@presensimu.com` / `admin123`
3. Test semua fitur:
   - Dashboard admin
   - Monitoring absensi
   - Jadwal kerja
   - Pengajuan cuti
4. Login sebagai karyawan: `karyawan@presensimu.com` / `karyawan123`
5. Test clock in/out
6. Cek riwayat absensi
