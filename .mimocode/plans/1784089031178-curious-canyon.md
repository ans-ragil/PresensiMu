# Railway Deployment - Database Setup & Login Test

## Status: Railway backend deployed ✅, database tables not yet created

## Steps

### 1. Create Database Tables
Run `prisma db push` to create all tables in PostgreSQL:

```bash
cd server
railway login
railway link
railway run npx prisma db push
```

This creates all 10 tables: Division, Position, Shift, User, Attendance, Schedule, LeaveRequest, Holiday, CompanyLocation, EmailSetting, Notification, CompanySetting.

### 2. Seed Default Data
```bash
railway run npx tsx prisma/seed.ts
```

Creates 4 accounts:
- admin@presensimu.com / admin123 (ADMIN)
- hr@presensimu.com / hr123 (HR)
- superadmin@presensimu.com / superadmin123 (SUPER_ADMIN)
- karyawan@presensimu.com / karyawan123 (EMPLOYEE)

### 3. Set Vercel Environment Variable
- Vercel dashboard → project Settings → Environment Variables
- Add: `VITE_API_URL` = `https://[service-name].up.railway.app/api`
- Click Redeploy

### 4. Test Login
Open Vercel app → login with any of the 4 accounts above

## Verification
- Health endpoint: `GET https://[service-name].up.railway.app/api/health` → `{"status":"ok"}`
- Login: POST `/api/auth/login` with admin credentials → returns tokens
- Vercel: page loads without 404 on refresh, login works
